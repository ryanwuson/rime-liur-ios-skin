# 鍵盤選擇彈窗功能實作總結

## 實作日期
2026-02-26

## 功能概述
在進階微調頁面新增鍵盤選擇彈窗功能，當使用者點擊「套用基礎外觀設定」按鈕時：
- 如果已經有開啟的鍵盤：顯示確認訊息（原有行為）
- 如果沒有開啟任何鍵盤：顯示選擇彈窗，讓使用者勾選要套用的鍵盤，同時開啟並套用設定

## 實作內容

### 1. Vue 數據結構（previewState）
新增以下屬性：
```javascript
showKeyboardSelectionModal: false,  // 鍵盤選擇彈窗顯示狀態
keyboardSelection: {                // 鍵盤選擇狀態（預設全選）
    keyboard26Chinese: true,
    numeric: true,
    symbolic: true,
    emoji: true
}
```

### 2. Vue 方法

#### 修改現有方法
- `applyBasicToAdvanced()`: 
  - 檢查是否有已開啟的鍵盤
  - 如果沒有，顯示選擇彈窗（而不是警告）
  - 如果有，顯示確認訊息（原有行為）

#### 新增方法
- `selectAllKeyboards()`: 全選所有鍵盤
- `deselectAllKeyboards()`: 取消選擇所有鍵盤
- `confirmKeyboardSelection()`: 確認選擇並執行以下操作：
  1. 收集選中的鍵盤
  2. 開啟選中鍵盤的進階微調開關
  3. 套用基礎外觀設定到選中的鍵盤
  4. 關閉彈窗
- `cancelKeyboardSelection()`: 取消選擇，關閉彈窗

### 3. HTML 結構
新增鍵盤選擇彈窗（使用 Teleport 到 body）：
- 使用現有的 modal-overlay 和 modal-container 樣式
- 4 個 checkbox 選項（26鍵、數字、符號、Emoji）
- 全選/全不選快捷按鈕
- 提示訊息：「選中的鍵盤將開啟進階微調並套用設定」
- 確定/取消按鈕

### 4. 樣式
- 複用現有的 modal 樣式（modal-overlay, modal-container, btn-modal 等）
- 自訂 checkbox 樣式（inline style）：
  - 選中時背景變為淺藍色 (#eff6ff)
  - Hover 效果
  - 使用 accent-color 設定 checkbox 顏色為藍色
- 暗色模式自動支援（透過現有的 .appearance-dark 樣式）

## 彈窗設計

### 視覺結構
```
┌─────────────────────────────────────────┐
│  套用基礎外觀設定（亮色模式/暗色模式）    │
├─────────────────────────────────────────┤
│  請選擇要套用的鍵盤：                    │
│                                          │
│  ☑ 26 鍵中英文鍵盤                       │
│  ☑ 數字鍵盤                              │
│  ☑ 符號鍵盤                              │
│  ☑ Emoji 鍵盤                            │
│                                          │
│  [全選] [全不選]                         │
│                                          │
│  💡 選中的鍵盤將開啟進階微調並套用設定    │
│                                          │
│  [取消] [確定]                           │
└─────────────────────────────────────────┘
```

### 互動行為
1. 預設全選所有鍵盤
2. 點擊 checkbox 或整個 label 區域可切換選擇狀態
3. 選中的項目背景變為淺藍色
4. Hover 時背景變為淺灰色
5. 全選/全不選按鈕可快速切換所有選項
6. 點擊「確定」：
   - 檢查是否至少選擇一個鍵盤
   - 開啟選中鍵盤的進階微調
   - 套用基礎外觀設定
   - 關閉彈窗
7. 點擊「取消」或彈窗外部：關閉彈窗，不執行任何操作

## 使用流程

### 情境 1：沒有開啟任何鍵盤
1. 使用者點擊「套用基礎外觀設定」按鈕
2. 顯示鍵盤選擇彈窗（預設全選）
3. 使用者可以：
   - 保持全選，直接點擊「確定」
   - 取消部分選項，只套用到特定鍵盤
   - 使用「全選」/「全不選」快速切換
4. 點擊「確定」後：
   - 選中的鍵盤自動開啟進階微調
   - 基礎外觀設定自動套用到選中的鍵盤
   - 彈窗關閉

### 情境 2：已經開啟部分鍵盤
1. 使用者點擊「套用基礎外觀設定」按鈕
2. 顯示確認訊息（原有行為）：
   ```
   ✓ 將套用基礎外觀設定（亮色模式）到以下鍵盤：
   • 26 鍵中英文鍵盤
   • 數字鍵盤
   ```
3. 使用者確認後，套用到已開啟的鍵盤

## 技術細節

### 響應式設計
- 彈窗寬度：max-width: 450px
- 在手機上自動適應寬度（90%）
- 使用 flexbox 確保元素對齊
- Checkbox 大小：18x18px（適合觸控）

### 暗色模式支援
- 自動繼承現有的 .appearance-dark 樣式
- 彈窗背景：#1c1c1e
- 文字顏色：white
- 輸入框背景：#2c2c2e

### 效能考量
- 使用 v-if 控制彈窗顯示（不渲染時不佔用 DOM）
- 使用 Teleport 將彈窗渲染到 body（避免 z-index 問題）
- 使用 @click.self 在彈窗外部點擊時關閉
